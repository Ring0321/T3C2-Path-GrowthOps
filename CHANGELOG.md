# Changelog

All notable changes are documented here. The project follows Semantic Versioning.

## [Unreleased]

## [0.2.0] - 2026-07-19

### Added

- Executable path graph with dependencies, hard deadlines, transferable assets, primary/secondary/backup roles, switching and rollback history.
- Context-preserving orchestration that returns separate profile snapshots and defers when a preregistered context gap is exceeded.
- Explicit service-effect study design, assignment-versus-receipt treatment semantics, missing-outcome strategy, temporal-order checks and claim-level gating.
- Evidence-backed VA gate requiring a documented invariance assessment, temporal split and frozen reference-model specification.
- Deterministic R01-R12 red-team runner, machine-readable results and a reason-required human override contract.
- Submission release manifest that distinguishes the canonical open-source conformance generator from the separate document design fixture.

### Changed

- Package and API version advanced to 0.2.0.
- Observational service estimates are limited to associational claims; synthetic exercises remain `SYNTHETIC_ONLY`.
- Numeric estimates from different synthetic data-generating processes are explicitly non-comparable.

### Added

- Executable specification, research boundaries and architecture baseline.
- Immutable domain contracts for consent, evidence, profiles, paths, tasks, service exposure, VA, SE, disputes, review tickets and audit events.
- Separate evidence, path and service clocks with expiry, crossover and intervention-version diagnostics.
- Algorithm A robust evidence-state updates with reliability, decay, duplicate-source and context-conflict handling.
- Algorithm B reproducible multi-path Monte Carlo simulation with hard-rule blocking and Pareto filtering.
- Algorithm C Safe-VOI task gating that prevents scores from overriding consent, reversibility, high-stakes and lower-cost alternatives.
- Algorithm D formative VA with measurement-invariance, reference-sample and interval-width stop rules.
- Algorithm E randomized ITT and AIPW service-effect estimators with target-trial, intervention-version and positivity checks.
- Algorithm F calibration, risk-coverage, group error/coverage/burden audits and selective publication gates.
- Capability-bounded evidence, path, task, governance, explanation and audit agents.
- Deterministic decision orchestrator, tamper-evident audit chain, synthetic CLI and versioned REST endpoint.
- Reproducible 1,200-row known-truth cohort, estimator report, integrity manifest and 12-case red-team registry.
- Model card, data card, claim register, validation protocol, threat model and GitHub quality/security automation.
