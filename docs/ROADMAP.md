# 研发路线图

本路线图把“继续开发”约束为能够被验证、被否定和被审计的版本增量。日期是目标窗口，不代表已取得真实学生或企业效果。

## v0.1 研究基线（已完成）

- 固化不可变领域对象、证据时钟、路径时钟和服务时钟。
- 实现动态证据状态、多路径数字孪生、Safe-VOI、学生增值、服务效果估计和选择性发布门。
- 建立最小权限智能体、多智能体编排、哈希链审计、CLI 与参考 API。
- 发布已知真值合成队列、数据卡、模型卡、红队案例和可重复 CI。

退出标准：测试覆盖率不低于 85%，静态检查、类型检查、构建、合成结果复现和依赖审计全部通过。

## v0.2 原型可用性与治理（下一阶段）

- 构建学生证据查看、更正、撤回和争议升级界面。
- 将路径硬约束与知识版本存储抽象为可替换适配器。
- 增加认证、最小权限授权、静态/传输加密接口和审计导出。
- 通过 12—15 名参与者的任务型可用性测试验证理解、纠错和退出能力。

退出标准：关键任务完成率达到预注册阈值；未通过时降级为顾问内部原型，不开放学生自主决策。

## v0.3 八周等待组试点（需伦理与数据授权）

- 预注册样本、主要指标、缺失处理、污染分析和停止规则。
- 分离学生成长 VA、随机分配 ITT 与观察性服务暴露分析。
- 记录模型版本、知识版本、人工修改和服务实际暴露。
- 输出置信区间、风险覆盖、公平性与学生负担指标。

退出标准：只在预设校准、安全、公平与数据质量门全部通过时扩大试点；否则停止对应功能或回到人工复核。

## v1.0 多校验证与产品化候选

- 进行跨学校、专业、年级和赛道的外部验证与漂移监测。
- 完成企业级身份、密钥、隔离、备份、恢复和事件响应。
- 接入飞书或其他 GrowthOps 工作流，但保持学生端入口可替换。
- 形成面向高校与服务组织的透明效果报告，不发布个体因果归因。

退出标准：外部验证、隐私影响评估、渗透测试、可访问性测试和人工治理机制均达到部署组织批准标准。

## 永久不进入路线图的用途

- 用画像、VA 或 SE 自动淘汰学生、限制教育机会或进行惩罚。
- 在无识别条件时宣称服务导致个体成功。
- 在规则过期、证据不足或路径硬约束未知时输出确定性成功概率。
- 将合成验证、企业宣传或前后测差异写成真实世界因果效果。
